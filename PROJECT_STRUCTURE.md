## Key Locations

### Backend Data (All in Backend/)

| What | Where |
|------|-------|
| Generated Personas | `Backend/data/personas/` |
| Test Scenarios | `Backend/data/scenarios/` |
| Execution Logs | `Backend/data/logs/` |
| Database | `Backend/database/persona_automation.db` |
| Configuration | `Backend/config/config.yaml` |

### Why Personas in Backend/data/?

**Personas are BACKEND RESOURCES because:**

1. ✅ Generated by Backend (PersonaGenerator LLM)
2. ✅ Stored in Backend database
3. ✅ Used by Backend agents for automation
4. ✅ Frontend only DISPLAYS them via API
5. ✅ Reduces data duplication
6. ✅ Single source of truth

**Data Flow:**
```
PersonaGenerator (Backend)
    ↓
Backend/data/personas/ (JSON files)
    ↓
Backend database (SQLite)
    ↓
Backend API endpoint
    ↓
Frontend (displays via dashboard)
```

## Separation of Concerns

### Backend Responsibilities
- ✅ Generate personas (LLM)
- ✅ Manage scenarios
- ✅ Execute automation
- ✅ Store all data
- ✅ Provide REST API
- ✅ Run agents & tools

### Frontend Responsibilities
- ✅ Display personas
- ✅ Show scenarios
- ✅ Trigger executions
- ✅ Display results
- ✅ Handle user input
- ✅ Dashboard UI

## Communication Flow

```
1. USER INTERACTION
   Frontend UI
   ↓
2. API REQUEST
   Frontend → Backend REST API
   (POST /api/personas, GET /api/personas, etc.)
   ↓
3. BACKEND PROCESSING
   Backend processes request
   Stores/retrieves from Backend/data/ and Backend/database/
   ↓
4. API RESPONSE
   Backend → Frontend (JSON)
   ↓
5. DISPLAY
   Frontend displays response
```

## Starting Both Services

```bash
# Terminal 1 - Backend
cd Backend
python src/main.py
# Runs on http://localhost:5000

# Terminal 2 - Frontend  
cd frontend
npm start
# Runs on http://localhost:3000
```

Frontend calls Backend API:
- `http://localhost:5000/api/...`

## Benefits of This Structure

✅ **Clear Separation** - Backend and Frontend are independent
✅ **Scalability** - Easy to scale each service separately
✅ **Maintainability** - Each service has clear responsibilities
✅ **Reusability** - Backend API can be used by multiple frontends
✅ **Testing** - Backend and Frontend can be tested independently
✅ **Deployment** - Can deploy Backend and Frontend separately
✅ **Data Organization** - Single source of truth (Backend/data/)

## No Floating Files

Before:
```
generated_personas/  ← OUTSIDE any folder (confusing)
src/                 ← OUTSIDE Backend (confusing)
config/              ← OUTSIDE Backend (confusing)
database/            ← OUTSIDE Backend (confusing)
```

After:
```
Backend/             ← All backend resources together
├── src/             ← Clean & organized
├── data/            ← Personas, scenarios, logs
├── database/        ← Database files
└── config/          ← Configuration
```
